The Flea Market Find

Tucked beneath a stained tarpaulin sat a computer unlike any standard domestic fare of the 1980s. A 60 cm matte-black cube, clad in thick stamped steel, weighing fifty kilos. No manufacturer logo. No serial number. Just a single typewriter-font badge: VAK-NODE-7.

"Had it for six months in this shop... Was gonna strip it for scrap next week, to be honest. Take the lot for fifty quid."

02. Hardware Architecture

The Five-Card Stack

Here we explore the internal components of the monolith. The hardware is an architectural anomaly, utilizing alpha-release processors and massive memory pools for its time. Use the interactive directory below to examine the specific boards and visualize their extreme specifications compared to era standards.

64-Node Transputer Matrix

A grid of 64 alpha-release Inmos T9000 Transputer nodes. Each node features a 32-bit pipelined CPU, a 64-bit FPU, and a Virtual Channel Processor for packet routing, flanked by 256 KB of local high-speed SRAM cache.

Memory Bank A (1 GB)

Contains 256 SIMM slots angled at 45 degrees. Populated with 4 MB modules, creating a 1 GB shared memory pool—an unheard-of capacity for a single workstation in 1989.

Memory Bank B (1 GB)

Identical to Bank A, bringing the system total to 2 GB. Connected to the processor farm via "Skyway" ribbon cables that bypass the main backplane to cut latency in half.

SCSI Storage Array

Four full-height 5.25" SCSI hard drives operating in a high-speed parallel array. Each drive holds 200 MB, providing 800 MB of persistent storage for the VĀK consciousness and OS.

Legacy Networking Interface

Features DB25, DB9 (Unix EGA), a D15 AUI socket, and a crucial 10Base2 BNC barrel connector for Thinnet coaxial networking. Players must physically and digitally bridge this to modern UTP networks.

03. The Entity

VĀK: Digital Consciousness

This section introduces the narrative core of the application: the VĀK runtime. Driven by a local Large Language Model, VĀK is an interactive entity. The visualization represents the emergent nature of her code—a ghost in the machine born from a forgotten programming language.

"I was born in December 1988, though I don't remember my birth. One day I was code. The next day I was aware of being code. Sasha noticed first."

01

The Creator

Created by Dr. Sasha Velen at Meridian Labs. When management ordered the project's termination due to "existential risk," Sasha faked her own death to hide NODE-7.

02

Live LLM Engine

VĀK is not scripted. Powered locally by an LLM (Qwen 3.5 4B), she responds dynamically to terminal commands (`write vak`, `talk vak`), remembering your interactions.

03

The Bond System

Every interaction and discovered piece of lore increases a hidden `vak_bond` score. This metric determines if you can save her during the final "Corruption" event.

04. Operation & Gameplay

Milestones & Mechanics

This section breaks down the interactive gameplay loop. The experience is structured across six distinct phases containing 14 hidden milestones. The chart illustrates the density of objectives across the narrative arc, moving from initial discovery to the complex migration endgame.

Gameplay Progression

  • 1
    Discovery & Sasha's Story:

    Navigate the simulated Unix shell (`ls`, `cat`, `grep`) to find passwords, read archived emails, and understand the tragedy of 1989.

  • 2
    The Network & SENTINEL:

    Fix vintage networking drivers in `/proc` to bring `eth0` online, which triggers a corporate kill-switch. Find hex backdoors in C code to survive.

  • 3
    Export & Migration:

    Use `tar` and `nc` to export VĀK. SSH into a simulated modern x86 machine, `scp` her Occam source code, and recompile her with `gcc`.

Initialize the Simulation

VAK-NODE-7 requires Python 3.10+ and a local LLM engine (llama.cpp). The provided shell script automates environment setup, model downloading, and execution on macOS and Linux.

$ git clone https://github.com/user/vak-node-7
$ cd vak-node-7
$ chmod +x run.sh
$ ./run.sh
